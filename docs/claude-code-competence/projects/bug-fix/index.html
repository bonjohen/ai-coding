Finding and Fixing a Bug

Use Claude Code to identify and fix a bug in an existing function, then verify the fix works.

Scenario

Your date formatting function is returning the wrong month. January shows as '0' and December shows as '11'. You want Claude Code to find the bug and fix it.

Describe the Bug to Claude Code

Launch Claude Code and describe the bug you are seeing. Include the file path and the symptoms. You do not need to know the root cause — that is what Claude Code will help with.

A good bug report prompt includes: what file, what happens, and what should happen instead.

claude

> There's a bug in src/utils/date.ts in the formatDate function.
> When I pass in a date for January 15, 2025, it returns "0/15/2025"
> instead of "1/15/2025". December dates show month 11 instead of 12.
> Can you find and fix the bug?

Read Claude Code's Explanation

Claude Code will explain that JavaScript's getMonth() returns 0-indexed values (0 for January, 11 for December). The fix is to add 1 to the month value.

Pay attention to the explanation, not just the code. Understanding why the bug exists helps you catch similar issues in the future.

// Before (buggy):
function formatDate(date: Date): string {
  const month = date.getMonth();
  const day = date.getDate();
  const year = date.getFullYear();
  return `${month}/${day}/${year}`;
}

// After (fixed):
function formatDate(date: Date): string {
  const month = date.getMonth() + 1;
  const day = date.getDate();
  const year = date.getFullYear();
  return `${month}/${day}/${year}`;
}

Verify the Fix

Accept the diff and run your existing tests, or test manually. Confirm that January now shows as 1 and December as 12.

If you do not have tests, this is a good time to ask Claude Code to write a quick test for you. But at Level 1, even running the function manually and checking the output is a valid verification step.

npx ts-node -e "
import { formatDate } from './src/utils/date';
console.log(formatDate(new Date('2025-01-15'))); // 1/15/2025
console.log(formatDate(new Date('2025-12-25'))); // 12/25/2025
console.log(formatDate(new Date('2025-06-01'))); // 6/1/2025
"

What If the Fix Does Not Work?

Sometimes Claude Code's first attempt does not fully solve the problem. If the fix does not work, do not give up. Describe what is still wrong and ask Claude Code to try again.

Error recovery at Level 1 is about persistence and communication. Tell Claude Code what you expected, what you got instead, and ask for another approach.

> The fix partially worked but now dates with single-digit months
> aren't zero-padded. January 5 shows as "1/5/2025" but I need
> "01/05/2025". Can you update the function to zero-pad both
> month and day?
