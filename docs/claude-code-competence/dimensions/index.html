Evaluation Dimensions

Cross-level dimensions used to evaluate competence across the framework.

Conceptual Understanding

How well the practitioner understands the underlying architecture of Claude Code — context windows, tool use, model behavior, and system constraints.

Why It Matters

Without accurate mental models, practitioners cannot predict tool behavior, diagnose failures, or make informed decisions about prompting strategy.

Assessment Questions

  • Can you explain how context window limits affect multi-file tasks?
  • Do you understand the difference between tool use and direct generation?
  • Can you describe what happens when Claude Code executes a bash command?
Assess this dimension →

Task Framing

The skill of breaking down work into well-scoped prompts with clear context, constraints, and success criteria that Claude Code can act on effectively.

Why It Matters

Task framing is the primary lever for output quality. Poor framing leads to wasted iterations and incorrect results regardless of tool capability.

Assessment Questions

  • Do you provide context about the codebase before asking for changes?
  • Can you decompose a large feature into sequential prompts?
  • Do you specify constraints and edge cases upfront?
Assess this dimension →

Verification Discipline

The practice of systematically reviewing, testing, and validating code produced by Claude Code before accepting it.

Why It Matters

AI-generated code can be subtly wrong. Without verification discipline, bugs accumulate silently and trust erodes over time.

Assessment Questions

  • Do you review every diff before accepting?
  • Do you run tests after each significant change?
  • Do you verify that generated code handles edge cases?
Assess this dimension →

Context Management

The ability to control what information Claude Code has access to, including file context, conversation history, and persistent configuration.

Why It Matters

Context is the model's working memory. Managing it well improves output quality and reduces token waste. Managing it poorly leads to hallucination and drift.

Assessment Questions

  • Do you use CLAUDE.md to persist project-level context?
  • Can you manage context window limits across long sessions?
  • Do you know when to start a fresh conversation vs. continue?
Assess this dimension →

Environment Design

The practice of configuring CLAUDE.md files, hooks, MCP servers, permissions, and project structure so that Claude Code operates more effectively by default.

Why It Matters

A well-designed environment reduces repetitive prompting, enforces standards automatically, and makes Claude Code more productive for everyone on the team.

Assessment Questions

  • Have you configured CLAUDE.md for your projects?
  • Do you use hooks to enforce pre-commit checks?
  • Have you set up MCP servers for external tool access?
Assess this dimension →

Workflow Scaling

The skill of designing workflows where multiple Claude Code instances collaborate, tasks are parallelized, and AI coding integrates into CI/CD pipelines.

Why It Matters

Scaling beyond single-session usage is where AI coding delivers order-of-magnitude productivity gains. Without workflow design, gains plateau at individual task speed.

Assessment Questions

  • Can you run multiple Claude Code agents on parallel tasks?
  • Have you integrated Claude Code into CI/CD pipelines?
  • Do you use headless/non-interactive mode for automation?
Assess this dimension →

Safety and Governance

The discipline of ensuring that AI-generated code is safe, reviewed, compliant with organizational standards, and auditable in production.

Why It Matters

As AI coding scales, the risk surface grows. Without governance, organizations face compliance failures, security vulnerabilities, and unauditable code changes.

Assessment Questions

  • Do you review AI-generated code for security vulnerabilities?
  • Is there an audit trail for AI-generated changes?
  • Do you have policies for what AI agents can and cannot do?
Assess this dimension →
