Glossary

Key terms and definitions used in the Claude Code Competence framework.

Agent

An AI system that can autonomously plan and execute multi-step tasks using tools, within defined boundaries.

Related: subagent, multi-agent

Hook

A user-configured shell command that runs automatically in response to Claude Code events such as tool calls or notifications.

Related: pre-commit, automation

Skill

A slash-command extension in Claude Code that provides specialized capabilities, invoked with /skill-name syntax.

Related: MCP, tool use

MCP (Model Context Protocol)

An open protocol that allows AI models to connect to external tools and data sources through standardized server interfaces.

Related: MCP server, tool use

Verification Loop

A systematic practice of reviewing, testing, and validating AI-generated output before accepting it into the codebase.

Related: verification discipline, testing

Context Window

The maximum amount of text (measured in tokens) that a language model can process in a single interaction.

Related: context management, token

Non-Interactive Mode

Running Claude Code without human interaction, using flags like --print or piped input, suitable for automation and CI/CD.

Related: headless, automation

Subagent

A Claude Code instance spawned by another agent to handle a subtask, operating with its own context window.

Related: agent, multi-agent

Governance

The organizational framework of policies, processes, and controls that ensure AI-generated code meets quality, security, and compliance standards.

Related: safety, audit trail

Review Automation

Using AI agents to automatically review pull requests, check for issues, and provide feedback as part of the development workflow.

Related: CI/CD, verification loop

CLAUDE.md

A markdown file placed in a project directory that provides persistent context, conventions, and instructions to Claude Code across sessions.

Related: context management, environment design

Headless Mode

Running Claude Code without a terminal UI, typically for automated or scripted workflows.

Related: non-interactive mode, automation
