Level 1

Operator

Uses Claude Code as a command-line tool with basic prompting and accepts output with minimal verification.

Definition

The Operator treats Claude Code as a smart autocomplete or command executor. Interactions are single-turn or loosely structured. The practitioner can install, launch, and prompt Claude Code but lacks a systematic approach to task framing or output verification.

At this level, the tool is useful but unpredictable. Results vary because prompts are vague, context is not managed, and there is no feedback loop beyond 'does it look right?'

Core Concepts

CLI Basics

Understanding how to install, launch, and interact with Claude Code from the command line.

Basic Prompting

The ability to express a request in natural language that Claude Code can act on.

Tool Output Reading

The ability to read and interpret Claude Code's responses, diffs, and tool invocations.

Model Awareness

A basic understanding that Claude Code is powered by a language model with capabilities and limitations, not a deterministic compiler.

Permission Model

Understanding that Claude Code asks for permission before executing commands, editing files, or accessing resources, and why those guardrails exist.

Core Skills

Install and Run

Can install Claude Code and start a session.

Observable evidence: Launches Claude Code from terminal without assistance.

Simple Prompting

Can write a natural language prompt that produces useful output.

Observable evidence: Asks Claude Code to write a function and gets a working result.

Accept or Reject Output

Can review a generated diff and decide whether to accept it.

Observable evidence: Reviews a diff and either accepts or asks for revision.

Tool Navigation

Can use slash commands (/help, /clear) and understand the basic tool palette.

Observable evidence: Uses /help to discover available commands without external documentation.

Basic Error Recovery

Can recognize when Claude Code has produced an error and retry with a different approach.

Observable evidence: Rephrases a prompt after getting an unhelpful response rather than giving up.

Observable Behaviors

  • Launches Claude Code and immediately starts prompting without setting context.
  • Accepts generated code without running tests.
  • Uses single-turn prompts for complex multi-step tasks.
  • Does not use CLAUDE.md or any persistent configuration.

Typical Failure Modes

  • Accepts incorrect code because the output 'looks right.'
  • Repeats the same vague prompt when results are poor instead of refining.
  • Loses context in long sessions without realizing it.
  • Does not understand why the same prompt produces different results.

Evaluation Tasks

Generate a Utility Function

Ask Claude Code to write a string utility function with edge cases.

Success Criteria

  • Function works correctly
  • Practitioner can explain what it does
  • Practitioner reviews the diff before accepting

Fix a Known Bug

Give Claude Code a file with a known bug and ask it to find and fix the issue.

Success Criteria

  • Bug is correctly identified
  • Fix is applied cleanly
  • Practitioner verifies the fix by running the code
