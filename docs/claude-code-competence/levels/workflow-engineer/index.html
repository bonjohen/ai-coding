Workflow Engineer

Designs multi-agent workflows, CI/CD integration, and automated development pipelines using Claude Code.

Definition

The Workflow Engineer operates at the system level. They design workflows where multiple Claude Code instances collaborate, tasks are parallelized across agents, and AI coding is integrated into continuous integration and deployment pipelines.

At this level, Claude Code is not just a tool — it is infrastructure. The practitioner thinks about orchestration, error handling, and workflow resilience.

Core Concepts

Multi-Agent Orchestration

Running multiple Claude Code instances on related but independent tasks, coordinating their work.

CI/CD Integration

Embedding Claude Code into automated pipelines for code generation, review, and testing.

Headless Operation

Using Claude Code in non-interactive mode for scripted and automated workflows.

Worktree Isolation

Using git worktrees to give each agent an isolated copy of the repository, preventing merge conflicts during parallel operation.

Output Quality Monitoring

Designing feedback loops that detect when automated agent output drifts below acceptable quality thresholds.

Core Skills

Workflow Orchestration

Can design and run multi-agent workflows with proper task decomposition.

Observable evidence: Runs parallel agents on independent tasks and merges results.

Pipeline Integration

Can embed Claude Code into CI/CD pipelines for automated code operations.

Observable evidence: CI pipeline includes Claude Code steps for code review or generation.

Workflow Error Handling

Can design workflows that handle failures gracefully.

Observable evidence: Automated workflows retry or escalate on failure rather than silently breaking.

Task Graph Design

Can model dependencies between agent tasks and determine which can run in parallel.

Observable evidence: Draws or documents a task dependency graph before launching parallel agents.

Token and Cost Management

Can estimate and manage token usage across multi-agent workflows to control costs.

Observable evidence: Sets token budgets for automated workflows and monitors actual usage.

Observable Behaviors

  • Runs multiple Claude Code instances on parallel branches.
  • Uses headless mode for batch operations.
  • Integrates Claude Code into PR review automation.
  • Designs retry and fallback logic for automated workflows.

Typical Failure Modes

  • Over-automates without sufficient verification checkpoints.
  • Creates workflows that are brittle and hard to debug.
  • Does not monitor automated agent output for quality drift.
  • Ignores security implications of autonomous code generation.

Evaluation Tasks

Multi-Agent Workflow Design

Design and execute a workflow where 2+ Claude Code agents work on parallel tasks with coordinated output.

Success Criteria

  • Tasks are correctly decomposed
  • Agents work in parallel
  • Results merge without conflicts

CI/CD Pipeline Integration

Add a Claude Code step to an existing CI pipeline that automatically reviews or generates code on pull requests.

Success Criteria

  • Pipeline runs without manual intervention
  • Agent output is reviewed before merge
  • Failures are reported clearly
