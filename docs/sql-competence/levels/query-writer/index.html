Level 1

Query Writer

Reads data from single tables using basic SELECT, WHERE, and ORDER BY. Treats SQL as a lookup tool.

Definition

The Query Writer can retrieve data from a database using a GUI or command-line client. They write single-table queries, apply simple filters, and sort results.

At this level, SQL is useful but results are unpredictable. The practitioner does not think about NULLs, duplicates, or performance.

Core Concepts

SELECT Basics

Understanding how to retrieve columns from a single table using SELECT, WHERE, and ORDER BY.

Data Types

Awareness that columns have types (VARCHAR, INTEGER, DATE) and that type mismatches cause errors.

NULL Awareness

Basic understanding that NULL means 'unknown' and behaves differently from empty strings or zero.

Row Filtering

Using WHERE clauses with comparison operators, LIKE, IN, and BETWEEN.

Result Sorting

Using ORDER BY to control the sequence of returned rows.

Core Skills

Basic SELECT

Can write a SELECT statement that returns specific columns from a table.

Observable evidence: Writes SELECT col1, col2 FROM table rather than SELECT *.

WHERE Filtering

Can filter rows using WHERE with basic operators.

Observable evidence: Applies WHERE conditions to narrow result sets.

Result Sorting

Can sort query results using ORDER BY.

Observable evidence: Adds ORDER BY to queries that need deterministic output order.

SQL Client Usage

Can connect to a database and execute queries using a GUI or CLI client.

Observable evidence: Opens a SQL client and runs queries without external help.

Error Reading

Can read a SQL error message and understand what went wrong.

Observable evidence: Fixes syntax errors after reading the error message.

Observable Behaviors

  • Opens a SQL client and types SELECT * FROM table to explore data.
  • Adds WHERE clauses by trial and error rather than intentional filter design.
  • Does not think about NULLs, duplicates, or row counts.
  • Cannot explain why a query returns zero rows.

Typical Failure Modes

  • Uses SELECT * in production code.
  • Ignores NULL values and gets wrong aggregate results.
  • Does not check row counts to validate query correctness.
  • Cannot join two tables without help.

Graduation Criteria

  • Writes correct multi-table INNER JOINs without assistance.
  • Uses GROUP BY with at least one aggregate function accurately.
  • Can explain the difference between WHERE and HAVING.
  • Writes basic INSERT, UPDATE, and DELETE statements safely.

Evaluation Tasks

Write a Filtered Query

Write a query that returns customer names and order dates for orders placed in the last 30 days, sorted by date descending.

Success Criteria

  • Uses explicit column names, not SELECT *.
  • WHERE clause correctly filters by date.
  • Results sorted by order date descending.
