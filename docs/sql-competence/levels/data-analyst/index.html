Level 2

Data Analyst

Constructs multi-table JOINs, aggregations, subqueries, and basic window functions for structured data analysis.

Definition

The Data Analyst uses SQL as a primary analysis tool. They can answer non-trivial business questions from a relational database.

They write queries other people can read, handle NULLs and duplicates, use CTEs for organization, and write basic window functions.

Core Concepts

JOIN Types

Understanding INNER, LEFT, RIGHT, and CROSS JOINs and when to use each.

Aggregation

Using GROUP BY with aggregate functions (COUNT, SUM, AVG, MIN, MAX) and HAVING.

Subqueries

Writing subqueries in WHERE, FROM, and SELECT clauses.

CTEs

Using Common Table Expressions (WITH clauses) to organize complex queries.

Window Function Basics

Using ROW_NUMBER, RANK, and basic OVER clauses for analytical queries.

Core Skills

Multi-Table JOINs

Can join three or more tables correctly.

Observable evidence: Writes a query joining orders, customers, and products.

Aggregation Queries

Can group and aggregate data to answer business questions.

Observable evidence: Computes monthly revenue totals with GROUP BY.

CTE Organization

Uses CTEs instead of nested subqueries for readability.

Observable evidence: Restructures a nested query into CTEs when it exceeds two levels.

NULL Handling

Handles NULLs deliberately using COALESCE, IS NULL, and NULL-safe comparisons.

Observable evidence: Uses COALESCE in aggregations to avoid incorrect results.

DML Operations

Can write safe INSERT, UPDATE, and DELETE statements.

Observable evidence: Adds WHERE clause to every UPDATE/DELETE and verifies row counts.

Observable Behaviors

  • Structures queries with CTEs instead of deeply nested subqueries.
  • Chooses between LEFT and INNER JOIN deliberately.
  • Checks row counts before and after joins to detect fan-out.
  • Formats SQL consistently for readability.

Typical Failure Modes

  • Creates accidental row multiplication from incorrect joins.
  • Uses GROUP BY but includes non-aggregated columns incorrectly.
  • Writes window functions without understanding PARTITION BY scope.
  • Does not validate query results against known data.

Graduation Criteria

  • Designs a simple normalized schema (3NF) from requirements.
  • Writes and reasons about transactions with explicit COMMIT/ROLLBACK.
  • Reads a basic query execution plan and identifies a missing index.
  • Writes a stored procedure or function in at least one dialect.

Evaluation Tasks

Write an Analytical Report Query

Write a query that shows the top 5 customers by total spending in each product category, using window functions.

Success Criteria

  • Correctly joins customers, orders, and products.
  • Uses window functions to rank within categories.
  • Handles ties appropriately.

Exercises

Multi-Table Analytical Query

Write a query joining three or more tables with aggregation.

intermediate

Window Function Ranking

Practice ROW_NUMBER and RANK for analytical queries.

Learning Emphasis

Practice complex joins on real schemas.

Learn window function patterns.

Study NULL semantics thoroughly.

Related Dimensions

  • Query Construction — Ability to write correct, readable, and efficient SQL.
  • Analytical Techniques — Proficiency with aggregation, window functions, and analysis patterns.
  • Data Modeling — Skill in designing relational schemas and entity relationships.
