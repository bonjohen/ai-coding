Competency Matrix

Full competency matrix mapping dimensions against the five levels.

Dimension L1: Writer L2: Analyst L3: Designer L4: Engineer L5: Architect
Query Construction
Single-table SELECT with basic WHERE and ORDER BY.
  • Writes SELECT * or explicit columns from one table.
  • Uses WHERE with basic operators.
  • Cannot yet join tables.
Multi-table JOINs, CTEs, and basic aggregation.
  • Joins multiple tables correctly.
  • Uses CTEs for readability.
  • Writes GROUP BY with HAVING.
Complex analytical queries and DML.
  • Advanced window functions.
  • Writes stored procedures.
  • Complex INSERT/UPDATE patterns.
Optimizer-aware query patterns.
  • Rewrites queries for better plans.
  • Avoids anti-patterns that prevent index use.
  • Uses query hints judiciously.
Defines query standards and anti-patterns for the organization.
  • Establishes query coding standards.
  • Reviews query patterns across teams.
  • Documents anti-patterns with examples.
Data Modeling
No schema design; works with existing tables.
  • Understands tables have columns and rows.
  • Cannot create tables from requirements.
  • No awareness of normalization.
Understands existing schemas and relationships.
  • Reads ER diagrams.
  • Understands foreign key relationships.
  • Can identify the join path between tables.
Designs normalized schemas from requirements.
  • Applies 1NF through 3NF.
  • Designs entity relationships.
  • Writes DDL with constraints.
Partitioned and performance-optimized schemas.
  • Denormalizes with reasoning.
  • Designs for specific query workloads.
  • Partitions large tables.
Enterprise canonical data models.
  • Designs cross-service data models.
  • Defines data ownership boundaries.
  • Evolves schemas across product cycles.
Analytical Techniques
Basic filtering only.
  • WHERE clause filtering.
  • No aggregation.
  • No window functions.
GROUP BY and basic window functions.
  • COUNT, SUM, AVG aggregations.
  • ROW_NUMBER, RANK.
  • Basic OVER clauses.
Advanced window functions and pivoting.
  • LAG, LEAD, NTILE.
  • Pivoting and unpivoting patterns.
  • Running totals and moving averages.
Performance-aware analytical patterns.
  • Optimizes analytical queries for large datasets.
  • Uses materialized views for heavy aggregations.
  • Designs efficient time-series queries.
Defines analytical patterns for the organization.
  • Establishes standard analytical query patterns.
  • Designs data warehouse schemas.
  • Evaluates analytical database technologies.
Performance & Optimization
Unaware of performance considerations.
  • Does not think about query cost.
  • No awareness of indexes.
  • No understanding of execution plans.
Surface-level index awareness.
  • Knows indexes exist and speed things up.
  • Cannot design an index strategy.
  • Does not read execution plans.
Reads execution plans and applies basic tuning.
  • Reads EXPLAIN output.
  • Identifies sequential vs index scans.
  • Creates single-column indexes.
Deep index strategy and concurrency tuning.
  • Designs composite and covering indexes.
  • Tunes statistics and autovacuum.
  • Manages lock contention.
Capacity planning and architectural trade-offs.
  • Forecasts performance at scale.
  • Designs for horizontal scaling.
  • Evaluates hardware and cloud configurations.
Data Integrity & Transactions
Basic NOT NULL awareness.
  • Knows some columns cannot be empty.
  • No understanding of constraints.
  • No transaction awareness.
Basic constraints and simple transactions.
  • Uses NOT NULL and UNIQUE.
  • Writes simple BEGIN/COMMIT blocks.
  • Basic understanding of foreign keys.
All constraint types and isolation levels.
  • Uses CHECK, FK, and UNIQUE constraints.
  • Understands isolation levels.
  • Designs transactional workflows.
Lock management and concurrency patterns.
  • Manages deadlocks and lock contention.
  • Selects isolation levels per transaction.
  • Implements optimistic locking patterns.
Distributed consistency and governance.
  • Designs cross-service consistency strategies.
  • Defines data integrity standards.
  • Handles eventual consistency trade-offs.
Tooling & Workflow
Ad-hoc GUI client usage.
  • Uses a SQL client GUI.
  • Saves queries as local files.
  • No version control for SQL.
Structured scripts and basic migrations.
  • Organizes SQL in script files.
  • Uses basic migration commands.
  • Version controls SQL files.
Migration tooling and automated workflow.
  • Uses Flyway, Liquibase, or similar.
  • Migrations are reversible.
  • SQL is reviewed before merge.
CI/CD integration and automated testing.
  • Migrations run in CI pipeline.
  • Automated regression tests for schemas.
  • Uses linters and formatters.
Toolchain standards and release governance.
  • Defines migration tooling standards.
  • Designs release processes.
  • Manages cross-environment promotion.
Collaboration & Standards
No standards; writes solo scripts.
  • No naming conventions.
  • No documentation.
  • Works in isolation.
Follows team conventions.
  • Follows established naming patterns.
  • Writes basic comments.
  • Participates in code review.
Contributes to conventions and documentation.
  • Proposes naming convention improvements.
  • Documents schema decisions.
  • Reviews others' migrations.
Establishes review processes.
  • Designs schema review workflows.
  • Creates documentation templates.
  • Mentors junior team members.
Owns organization-wide data governance.
  • Defines cross-team data standards.
  • Leads data governance initiatives.
  • Manages data contracts between services.
