SQL Competence Framework

A five-level model of SQL proficiency measured across seven cross-cutting dimensions, from basic querying to enterprise data architecture.

This framework defines SQL competence as measurable growth across seven dimensions that each capture a distinct aspect of effective database work.

Every practitioner — from someone writing their first SELECT to someone designing enterprise-scale data architectures — operates across all seven dimensions simultaneously.

The Seven Dimensions

Each dimension is independent: a practitioner can be strong in Query Construction but weak in Collaboration & Standards.

Dimension

Query Construction

Ability to write correct, readable, and efficient SQL.
Dimension

Data Modeling

Skill in designing relational schemas and entity relationships.
Dimension

Analytical Techniques

Proficiency with aggregation, window functions, and analysis patterns.
Dimension

Performance & Optimization

Understanding of index design, execution plans, and query tuning.
Dimension

Data Integrity & Transactions

Command of constraints, ACID properties, and transactional design.
Dimension

Tooling & Workflow

Use of SQL clients, migrations, version control, and CI/CD.
Dimension

Collaboration & Standards

Adherence to naming conventions, documentation, and data governance.

The Five Levels

Each level represents a recognizable, observable mode of working with SQL. Movement between levels is a shift in how all seven dimensions are engaged together.

Level 1

Query Writer

Reads data using basic SELECT, WHERE, and ORDER BY.
Level 2

Data Analyst

Constructs multi-table JOINs, aggregations, and window functions.
Level 3

Schema Designer

Designs normalized schemas, manages transactions and constraints.
Level 4

Performance Engineer

Designs index strategies, optimizes queries, manages concurrency.
Level 5

Database Architect

Architects enterprise data systems, governance, and organizational strategy.

How to Use This Framework

Use the Self-Assessment to score yourself across all seven dimensions. Use the Competency Matrix to compare your weakest dimensions against your strongest.

Use the level detail pages for graduation criteria and evaluation tasks. Use the Study Guide to generate a prioritized reading list.
