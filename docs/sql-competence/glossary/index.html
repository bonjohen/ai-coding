Glossary

Key terms and definitions used in the SQL Competence framework.

CTE (Common Table Expression)

A named temporary result set defined with a WITH clause that exists for the duration of a single query. Improves readability and allows recursive queries.

Related: Subquery, Window Function

Normalization

The process of organizing database tables to reduce redundancy and dependency. Normal forms (1NF, 2NF, 3NF, BCNF) define progressive levels of structural integrity.

Related: Denormalization, Foreign Key

Denormalization

The deliberate introduction of redundancy into a normalized schema to improve read performance at the cost of write complexity.

Related: Normalization, Index

Index

A data structure that improves the speed of data retrieval at the cost of additional storage and slower writes. Common types include B-tree, hash, and GIN indexes.

Related: Execution Plan, Composite Index

Composite Index

An index on multiple columns. Column order matters — the index is most effective when queries filter on leftmost columns first.

Related: Index, Covering Index

Covering Index

An index that contains all columns needed by a query, allowing the database to satisfy the query from the index alone without accessing the table.

Related: Composite Index, Index

Execution Plan

The database engine's strategy for executing a query, showing the sequence of operations (scans, joins, sorts) and their estimated costs. Viewed via EXPLAIN.

Related: Index, Sequential Scan

Sequential Scan

A table scan that reads every row. Often indicates a missing index, but can be optimal for small tables or queries that need most rows.

Related: Execution Plan, Index

ACID

Atomicity, Consistency, Isolation, Durability — the four properties that guarantee reliable transaction processing in a relational database.

Related: Transaction, Isolation Level

Transaction

A sequence of SQL operations executed as a single unit of work. Either all operations succeed (COMMIT) or none take effect (ROLLBACK).

Related: ACID, Isolation Level

Isolation Level

Controls how concurrent transactions see each other's changes. Standard levels: READ UNCOMMITTED, READ COMMITTED, REPEATABLE READ, SERIALIZABLE.

Related: ACID, Transaction, MVCC

MVCC (Multi-Version Concurrency Control)

A concurrency control method where each transaction sees a snapshot of the database, allowing readers and writers to operate without blocking each other.

Related: Isolation Level, Transaction

Window Function

A function that performs a calculation across a set of rows related to the current row, using an OVER clause. Unlike GROUP BY, does not collapse rows.

Related: CTE (Common Table Expression), Partition

Partition

In window functions: the subset of rows the function operates on (PARTITION BY). In tables: a physical division of a large table into smaller segments for performance.

Related: Window Function, Index

Foreign Key

A constraint that enforces referential integrity between two tables by requiring values in one column to match values in another table's primary key.

Related: Normalization, Constraint

Constraint

A rule enforced by the database on column values. Types include NOT NULL, UNIQUE, CHECK, PRIMARY KEY, and FOREIGN KEY.

Related: Foreign Key, Normalization

Subquery

A query nested inside another query. Can appear in WHERE, FROM, or SELECT clauses. Correlated subqueries reference the outer query and execute per row.

Related: CTE (Common Table Expression)

Migration

A version-controlled script that applies a specific schema change to a database. Typically includes both an 'up' (apply) and 'down' (rollback) operation.

Related: DDL

DDL (Data Definition Language)

SQL statements that define or modify database structure: CREATE, ALTER, DROP, TRUNCATE. Distinct from DML which manipulates data.

Related: Migration, DML

DML (Data Manipulation Language)

SQL statements that read or modify data: SELECT, INSERT, UPDATE, DELETE. Distinct from DDL which modifies structure.

Related: DDL, Transaction

Data Contract

A formal agreement between services or teams about the structure, semantics, and guarantees of shared data. Prevents breaking changes from propagating.
