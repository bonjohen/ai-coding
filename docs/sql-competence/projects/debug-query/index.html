Level 1

Debugging a Broken Query

Find and fix errors in a SQL query by reading error messages.

Scenario

A colleague wrote a query that doesn't work. They get an error but can't figure out what's wrong. You need to read the error, identify the problem, and fix it.

Read the Error Message

The broken query and its error message are your starting point. Read the error carefully — SQL error messages usually tell you exactly what's wrong and where.

Don't just scan the query visually. Match the error message to the specific part of the query it references.

-- The broken query:
SELECT first_name last_name, email
FROM customer
WHERE signup_date = '2025-03-15'
ORDER BY last_name

-- Error: relation "customer" does not exist
-- (table is actually named 'customers')

Fix the Errors and Verify

This query has three errors: wrong table name ('customer' vs 'customers'), missing comma between first_name and last_name, and missing semicolon. Fix them all.

After fixing, always run the query and verify the results make sense.

-- Fixed query:
SELECT first_name, last_name, email
FROM customers
WHERE signup_date = '2025-03-15'
ORDER BY last_name;
