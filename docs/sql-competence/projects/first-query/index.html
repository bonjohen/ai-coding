Level 1

Your First Filtered Query

Write a query to retrieve specific data from a single table with filtering and sorting.

Scenario

You have access to a customer database. Your manager asks: 'How many customers signed up in March 2025, and can you list them alphabetically?' You need to write a query to answer this.

Connect and Explore the Table

First, connect to the database and understand the table structure. Before writing any query, look at what columns are available.

Always start by understanding the data. Running a quick exploratory query saves time later.

-- See what columns exist
SELECT column_name, data_type
FROM information_schema.columns
WHERE table_name = 'customers';

-- Peek at a few rows
SELECT * FROM customers LIMIT 5;

Write the Filtered Query

Now write the query with explicit column names (not SELECT *), a WHERE clause for the date range, and ORDER BY for alphabetical sorting.

Notice we select specific columns, not everything. This is a good habit even for simple queries.

SELECT first_name, last_name, email, created_at
FROM customers
WHERE created_at >= '2025-03-01'
  AND created_at < '2025-04-01'
ORDER BY last_name, first_name;

Answer the Count Question

Your manager also asked 'how many.' A separate COUNT query gives the answer clearly.

At Level 1, running a separate count query is fine. Later you will learn to combine these.

SELECT COUNT(*) AS march_signups
FROM customers
WHERE created_at >= '2025-03-01'
  AND created_at < '2025-04-01';
