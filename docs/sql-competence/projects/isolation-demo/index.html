Level 4

Isolation Level Experiment

Demonstrate concurrency anomalies and how isolation levels prevent them.

Scenario

You need to teach your team about isolation levels. Set up a hands-on demonstration showing a non-repeatable read and how REPEATABLE READ prevents it.

Set Up the Demonstration

Open two database sessions (Session A and Session B). Create a simple test table.

CREATE TABLE account_balance (
  account_id INT PRIMARY KEY,
  balance DECIMAL(10,2) NOT NULL
);
INSERT INTO account_balance VALUES (1, 1000.00);

Demonstrate the Anomaly

In READ COMMITTED (default), Session A reads the balance, Session B updates and commits, Session A reads again and sees a different value.

Then repeat with REPEATABLE READ to show the anomaly is prevented.

-- Session A (READ COMMITTED):
BEGIN;
SELECT balance FROM account_balance WHERE account_id = 1;
-- Returns: 1000.00

-- Session B (in parallel):
UPDATE account_balance SET balance = 500.00 WHERE account_id = 1;
COMMIT;

-- Session A reads again:
SELECT balance FROM account_balance WHERE account_id = 1;
-- Returns: 500.00  <-- non-repeatable read!
COMMIT;

-- Now repeat with REPEATABLE READ:
-- Session A:
BEGIN ISOLATION LEVEL REPEATABLE READ;
SELECT balance FROM account_balance WHERE account_id = 1;
-- Returns: 1000.00 (after resetting)

-- Session B updates and commits...

-- Session A reads again:
SELECT balance FROM account_balance WHERE account_id = 1;
-- Still returns: 1000.00  <-- consistent!
COMMIT;
