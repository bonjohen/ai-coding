Level 2

Monthly Revenue Report

Write a multi-table analytical query with JOINs, aggregation, and a CTE.

Scenario

The finance team needs a monthly revenue report broken down by product category. You need to join orders, order items, and products, then aggregate by month and category.

Plan the Join Path

Before writing, identify the tables and how they connect. Orders have order_items, order_items reference products, products belong to categories.

Draw the join path: orders -> order_items -> products -> categories. Each join is through a foreign key.

-- Join path:
-- orders.id = order_items.order_id
-- order_items.product_id = products.id
-- products.category_id = categories.id

Write the Query Using a CTE

Use a CTE to compute the line-item revenue first, then aggregate by month and category in the outer query.

Check row counts at each step. After the JOIN, verify no accidental row multiplication by comparing against expected order_items count.

WITH line_revenue AS (
  SELECT
    DATE_TRUNC('month', o.order_date) AS month,
    c.name AS category,
    oi.quantity * oi.unit_price AS line_total
  FROM orders o
  JOIN order_items oi ON oi.order_id = o.id
  JOIN products p ON p.id = oi.product_id
  JOIN categories c ON c.id = p.category_id
)
SELECT
  month,
  category,
  SUM(line_total) AS revenue,
  COUNT(*) AS line_items
FROM line_revenue
GROUP BY month, category
ORDER BY month DESC, revenue DESC;
