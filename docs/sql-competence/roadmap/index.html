SQL Learning Roadmap

A structured path through the SQL Competence framework, with recommended activities at each stage.

Learning Stages

Level 1

Stage 1: SQL Fundamentals

Build a solid foundation with single-table queries, basic filtering, and sorting.

Suggested Activities

  • Complete an interactive SQL tutorial (e.g., SQLBolt, W3Schools SQL).
  • Practice 20+ basic queries against a sample database.
  • Learn to read SQL error messages and fix syntax errors independently.

Example Projects

  • Your First Filtered Query
  • Debugging a Broken Query
Practice Quiz Self-Assessment
Level 2

Stage 2: Data Analysis with SQL

Advance to multi-table queries, aggregations, and analytical patterns.

Suggested Activities

  • Work through JOIN exercises with 3+ table schemas.
  • Learn window functions through progressive examples.
  • Practice writing CTEs to organize complex analytical queries.

Example Projects

  • Monthly Revenue Report
  • Top Customers by Region
Practice Quiz Self-Assessment
Level 3

Stage 3: Schema Design & Integrity

Learn to design schemas, enforce integrity, and manage database changes.

Suggested Activities

  • Design a schema from a feature specification and write DDL.
  • Set up a migration tool and write reversible migration scripts.
  • Practice reading execution plans for queries you write.

Example Projects

  • Design a Room Booking Schema
  • Execution Plan Analysis
Practice Quiz Self-Assessment
Level 4

Stage 4: Performance & Optimization

Develop deep skills in query optimization, index design, and concurrency.

Suggested Activities

  • Analyze slow query logs and propose optimization strategies.
  • Design composite indexes for real query workloads.
  • Study isolation levels with hands-on concurrency experiments.

Example Projects

  • Designing an Index Strategy
  • Isolation Level Experiment
Practice Quiz Self-Assessment
Level 5

Stage 5: Data Architecture & Governance

Lead data architecture decisions and establish organizational standards.

Suggested Activities

  • Design a multi-service data architecture with clear ownership boundaries.
  • Author SQL standards and review guidelines for your team.
  • Evaluate distributed database options for a real workload.

Example Projects

  • Multi-Service Data Architecture
  • Establishing Data Governance
